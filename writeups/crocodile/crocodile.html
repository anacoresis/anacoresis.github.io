HTB Machine: Crocodile

OS · Linux Difficulty · Very Easy (Tier 0) July 2026 Target · 10.129.1.15

Step 1 · Reconnaissance & Scanning

Run an initial nmap scan to discover open ports and determine the versions of running services:

sudo nmap -sV -sC 10.129.1.15

Key findings

  • Port 21 (FTP): Open and active. (Port 21 is the standard FTP port.)
  • Port 80 (HTTP): Open, web server hosting a web application.

Step 2 · FTP Exploitation

Attempting a standard CLI login returned a specific restriction message:

530 This FTP server is anonymous only.

Log in with the anonymous username and a blank password to bypass the authentication check:

ftp 10.129.1.15
# Username: anonymous
# Password: <Press Enter>

Step 3 · Data Exfiltration

Inside the FTP server, two interesting credential files were located. Use the get command to download them locally:

ftp> get allowed.userlist
ftp> get allowed.userlist.passwd

Reading the downloaded files revealed four potential usernames and four potential passwords:

Step 4 · Web Directory Brute-Forcing

Perform directory busting with gobuster on the HTTP service to find hidden paths and login portals:

gobuster dir -u http://10.129.1.15 -w /usr/share/wordlists/dirb/common.txt

Key finding

Found the /dashboard directory, which hosts the main administrative login portal.

Step 5 · Credential Stuffing & Flag Retrieval

Navigate to the login page found via directory busting. Manually test the exfiltrated usernames against the discovered passwords. Testing the admin user against the password list produced a successful login, granting access to the dashboard where the flag was recovered.
